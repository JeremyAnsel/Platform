BaseBriefing Class

[Serializable] public abstract class BaseBriefing
Base class for Briefings. Contains values that are shared among all briefing types. Class is Serializable to allow copy/paste functionality.

Properties

public string[] BriefingString
protected string[] _briefingStrings
Gets the Title and Caption strings. Length is determined by the derivative class

public string[] BriefingTag
protected string[] _briefingTags
Gets the Text Tags to be used on the map. Length is determined by the derivative class

public byte[] Events
protected byte[] _events
Gets the raw briefing event data. Although Events is a byte array, briefing values are actually shorts. Length is determined by the derivative class

public short EventsLength
Gets the number of int16 values in Events[]

public short Length
Gets or sets the duration of the Briefing, in Ticks. The X-wing series uses Ticks instead of seconds, TicksPerSecond is defined by the derivative class

protected MissionFile.Platform _platform
Internal Platform designation

public short StartLength
Gets the number of int16 values in Events[] at time = 0

public short Unknown1
protected short _unknown1
Gets or sets the unknown value. Briefing offset 0x02, between Length and StartLength. Defaults to 0

Enumerations

public enum EventType : byte { None, PageBreak = 3, TitleText, CaptionText, MoveMap, ZoomMap, ClearFGTags, FGTag1, FGTag2, FGTag3, FGTag4, FGTag5, FGTag6, FGTag7, FGTag8, ClearTextTags, TextTag1, TextTag2, TextTag3, TextTag4, TextTag5, TextTag6, TextTag7, TextTag8, XwaNewIcon, XwaShipInfo, XwaMoveIcon, XwaRotateIcon, XwaChangeRegion, EndBriefing = 0x22 }

Converter Class

public static class Converter
Object for Mission Platform conversions. Primarily handles downgrading of platforms, due to existing utilities for upgrading.
Converted files will use same MissionPath with platform included ("test.tie" to "test_Xvt.tie")

Methods

public static Tie.Mission XvtBopToTie(Xvt.Mission miss)
Downgrades XvT and BoP missions to TIE95. Returns the downgraded mission. G/PLT, SHPYD, REPYD and M/SC craft will have their indexes changed to reflect IDMR TIE95 Ships patch numbering. Triggers will update.
FG.Radio is not converted, since TIE behaviour is different
Maximum FG.Formation value of 12 allowed
For Triggers, maximum Trigger index of 24, maximum TriggerType of 9, Amounts will be adjusted as 66% to 75%, 33% to 50% and "each" to 100%
Maximum Abort index of 5
Maximum FG.Goal Amount index of 6, 75% converted to 100%, 25% to 50%. First three XvT Goals will be used as Primary, Secondary and Bonus goals. Bonus points will be scaled appropriately. Goals only used if set for Team[0] and Enabled
First two Arrival triggers used, first Departure trigger used. First three Orders used. All standard WPs and first Briefing WP used.
For Messages, first two triggers used.
For the Briefing, entire thing should be able to be used unless the original actually uses close to 200 commands (yikes). There is a conversion on the Zoom factor, this is a legacy factor from my old Converter program, I don't remember why.
Primary Global goals used, XvT Secondary goals converted to Bonus goals. Prevent goals ignored
Team[0] EndOfMissionMessages used, Teams[2-6] Name and Hostility towards Team[0] used for IFF
BriefingQuestions generated using MissionSucc/Fail/Desc strings. Flight Officer has a single pre-mission entry for the Description, two post-mission entries for the Success and Fail. Line breaks must be entered manually
Filename will end in "_TIE.tie"
miss - XvT/BoP mission to convert
Exceptions
System.ArgumentException - Properties incompatable with TIE95 were detected in miss

public static Xvt.Mission XwaToXvtBop(Xwa.Mission miss, bool bop)
Downgrades XWA missions to XvT and BoP. Returns the downgraded mission. Maximum CraftType of 91. Triggers will update.
For Triggers, maximum Trigger index of 46, maximum TriggerType of 23, Amounts will be adjusted as "each special" to "100% special"
Only Start and Hyp WPs converted, manual placement for WP1-8 required.
For the Briefing, first 32 strings and text tags are copied, events are ignored (due to using icons instead of Craft)
Filename will end in "_XvT.tie" or "_.BoP.tie"
miss - XWA mission to convert
bop - Determines if mission is to be converted to BoP instead of XvT
Exceptions
System.ArgumentException - Properties incompatable with XvT/BoP were detected in miss

public static Tie.Mission XwaToTie(Xwa.Mission miss)
Downgrades XWA missions to TIE95. G/PLT, SHPYD, REPYD and M/SC craft will have their indexes changed to reflect IDMR TIE95 Ships patch numbering. Triggers will update.
FG.Radio is not converted, since TIE behaviour is different
Maximum FG.Formation value of 12 allowed
For Triggers, maximum Trigger index of 24, maximum TriggerType of 9, Amounts will be adjusted as 66% to 75%, 33% to 50% and "each" to 100%
Maximum Abort index of 5

First two Arrival triggers used, first Departure trigger used. First three Region 1 Orders used, max index of 38.
Only Start and Hyp WPs converted, manual placement for WP1-8 required.
For Messages, first two triggers used.
For the Briefing, first 16 strings and text tags are copied, events are ignored (due to using icons instead of Craft)
Primary Global goals used, XWA Secondary goals converted to Bonus goals. Prevent goals ignored
Team[0] EndOfMissionMessages used, Teams[2-6] Name and Hostility towards Team[0] used for IFF
